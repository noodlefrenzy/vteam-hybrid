# Quickstart Backlog -- greeter

Generated by `/quickstart` on 2026-03-15. Product owner: Pat. Sprint planner: Grace.

## Items

- [x] **GRT-1: Basic greet command** (S)
  Accept a `--name` flag and print a greeting. Default to "World" when no name is given. Raise an error for empty string input.
  _Acceptance criteria:_ `greeter greet` prints "Hello, World!" / `greeter greet --name Alice` prints "Hello, Alice!" / `greeter greet --name ""` exits with error.

- [ ] **GRT-2: Farewell command** (S)
  Add a `farewell` subcommand that prints "Goodbye, {name}!" with the same defaulting and validation as `greet`.
  _Acceptance criteria:_ Same patterns as GRT-1 but with "Goodbye" instead of "Hello."

- [ ] **GRT-3: Greeting log** (M)
  Record each greeting to a local JSON file (`~/.greeter/history.json`). Add a `history` subcommand that prints the last 10 greetings with timestamps.
  _Acceptance criteria:_ After running `greet` three times, `greeter history` shows all three with ISO timestamps. File is created on first use. Corrupted file is backed up and recreated.

## Notes

- All items are independent -- no ordering dependency between GRT-2 and GRT-3.
- GRT-3 is sized M because it involves file I/O, error handling for corrupt state, and a new subcommand. Architecture Gate not required (no new patterns -- standard file I/O).
